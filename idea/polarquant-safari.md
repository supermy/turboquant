###### 抽象的

我们介绍**PolarQuant**，一种大型语言模型（LLM）的训练后权重量化方法，它利用神经网络权重的分布结构来实现近乎无损的压缩。PolarQuant分三个阶段运行：（1）块状归一化到单位超球体，（2）沃尔什-哈达达德旋转将坐标转换为近似高斯随机变量，以及（3）量化与高斯分布匹配的重心。我们的消融表明，仅Hadamard旋转就占**了质量改进的98%**，将Qwen3.5-9B的困惑从6.90（absmax Q5）减少到6.40（Δ=+0.03来自FP16），使其 *几乎无损，没有任何校准数据* 。此外，PolarQuant是下游INT4量化器的有效预处理步骤：PolarQuant Q5由torchao INT4去量化和重新量化，实现了直接absmax INT4的6.68的困惑度，而6.56.68，同时在6.5 GB VRAM下保持43.1 tok/s的吞吐量。代码和模型是公开的。

## 1介绍

在消费者和边缘硬件上部署大型语言模型需要积极的重量压缩。FP16中的90亿参数模型大约需要18 GB的GPU内存，超过了大多数消费者GPU的容量。量化到4位将其减少到大约5-6 GB，可以在从NVIDIA RTX台式机GPU到Apple Silicon笔记本电脑等设备上进行推理。

最简单、部署最广泛的量化方案是 *absmax* （绝对最大值）量化[[10](https://arxiv.org/html/2603.29078v1#bib.bib20 "神经网络的量化和训练，用于高效的整数算术推理")]，线性映射值[−α，α]到整数代码，其中α=马克斯⁡|W我|。虽然在计算上微不足道，但当权重分布不均匀时，absmax可以被证明是次优的：它在很少发生的离群值上浪费了宝贵的代码簿条目，并将量化误差集中在高密度中心区域。

更复杂的方法，如GPTQ** **[[8](https://arxiv.org/html/2603.29078v1#bib.bib1 "GPTQ：生成预训练变压器的准确训练后量化")]和AWQ** **[[11](https://arxiv.org/html/2603.29078v1#bib.bib2 "AWQ：用於裝置上LLM壓縮和加速的啟動感知重量量化")]，分别通过基于校准的黑森校正或激活感知缩放来缓解这个问题。正常浮动（NF4）[[7](https://arxiv.org/html/2603.29078v1#bib.bib3 "QLoRA：量化语言模型的高效微调")]采用分布方法，为正态分布权重设计最佳代码集。QuIP#** **[[6](https://arxiv.org/html/2603.29078v1#bib.bib4 "QuIP#：更好的LLM量化与hadamard不连贯和晶格代码簿")]应用随机不相干处理来改善量化边界。最近，TurboQuant** **[[3](https://arxiv.org/html/2603.29078v1#bib.bib5 "TurboQuant：具有接近最佳失真率的在线矢量量化")]表明，通过哈达马变换归一化和旋转KV缓存向量可以产生可实现最佳量化的高斯坐标。

在这项工作中，我们将极性量化概念从KV缓存压缩适应并扩展到*权重*压缩。我们的关键见解是，在块归一化和哈达马德旋转后，LLM权重块由i.i.d.标准正态随机变量很好地近似，其中MSE最优量化器通过Lloyd-Max算法在分析上已知[[13](https://arxiv.org/html/2603.29078v1#bib.bib8 "PCM中的最小二乘量化")，[14](https://arxiv.org/html/2603.29078v1#bib.bib9 "最小失真量化")]。这产生了一个量化方案，即：

1. **1.**
   **近乎無損。**仅哈达马德旋转就能将量化误差降低98%，将Q5的困惑感带入+0.03FP16（6.40 vs 6.37）没有任何校准数据。
2. **2.**
   **简单。**核心算法是单一矩阵乘法（H128B^我）每个块，没有梯度计算，没有迭代优化，也没有校准数据。Hadamard矩阵是确定性的和自逆的。
3. **3.**
   **可作曲的。**PolarQuant作为与任何下游量化器兼容的预处理步骤。我们证明了PolarQuant Q5去量化权重，当torchao INT4重新量化时[[16](https://arxiv.org/html/2603.29078v1#bib.bib16 "Torchao：PyTorch架构优化")]，在相同的速度和内存下，比直接absmax INT4的困惑度低0.12分。
4. **4.**
   **高效。**哈达玛德矩阵是它自己的逆数（HH⊤=我），不需要额外的存储空间。去量化在模型載入時間增加了大約8秒，零執行時間開銷。

我们在Qwen3.5-9B上评估PolarQuant** **[[17](https://arxiv.org/html/2603.29078v1#bib.bib22 "Qwen3技术报告")]并展示最先进的困惑-压缩权衡。结合AWQ，PolarQuant在WikiText-2上实现了6.43的困惑度[[15](https://arxiv.org/html/2603.29078v1#bib.bib14 "指针向子混合模型")]，只是+0.06在FP16以上，在3.6×压缩。在Apple Silicon（Mac mini M4）上，PolarQuant在4.8 GB内存下实现19.7 tok/s，复杂度为6.90。

## 2背景和相关工作

### 2.1Absmax量化

给定一个权重张量W分区成块{B我}大小D，absmax量化计算尺度S我=马克斯J⁡|B我，J|并将每个元素映射到最接近的整数{−2B−1，...，2B−1−1}：

|  | 问我，J=圆形的（B我，JS我⋅（2B−1−1））。 |  | **(1)** |
| - | ------------------------------------------- | - | ------------- |

这种方法假定分布均匀[−S我，S我]，这与经验观察到的近高斯权重分布不匹配[[4](https://arxiv.org/html/2603.29078v1#bib.bib21 "用于快速部署的卷积网络的训练后4位量化")]。

### 2.2GPTQ

GPTQ** **[[8](https://arxiv.org/html/2603.29078v1#bib.bib1 "GPTQ：生成预训练变压器的准确训练后量化")]使用近似的二阶（Hessian）信息进行分层量化，迭代量化列，并通过最佳脑外科医生框架补偿剩余列中的误差。GPTQ取得了强大的效果，但需要校准数据，对于大型模型来说计算成本很高。

### 2.3AWQ：激活感知权重量化

AWQ** **[[11](https://arxiv.org/html/2603.29078v1#bib.bib2 "AWQ：用於裝置上LLM壓縮和加速的啟動感知重量量化")]观察到一小部分重量通道不成比例地影响了输出质量，因为它们对应于大的激活幅度。AWQ从校准激活中计算每个通道的缩放系数，保护重要通道免受量化误差的影响。我们表明，AWQ补充了PolarQuant：AWQ在*通道*上运行，而PolarQuant在*块*上运行，它们的组合产生了近乎无损的压缩。

### 2.4正常浮动（NF4）

Dettmers等人。[[7](https://arxiv.org/html/2603.29078v1#bib.bib3 "QLoRA：量化语言模型的高效微调")]引入了NormalFloat，这是一种量化水平在标准正态分布的分位数域中均匀间隔的数据类型。当每个量化箱具有相等的概率时，NF4在信息理论上是最优的。PolarQuant在两个关键方面有所不同：（1）我们使用最小化MSE而不是最大化熵的*Lloyd-Max*重心，（2）我们通过哈达玛德旋转明确地将权重转换为高斯，而不是先验地假设高斯性。

### 2.5QuIP和QuIP#

奎普[[5](https://arxiv.org/html/2603.29078v1#bib.bib13 "QuIP：具有保证的大型语言模型的2位量化")]引入了权重量化的不相干处理，表明随机正交旋转提高了量化边界。QuIP#** **[[6](https://arxiv.org/html/2603.29078v1#bib.bib4 "QuIP#：更好的LLM量化与hadamard不连贯和晶格代码簿")]用随机哈达玛德变换（RHT）和2位量化的晶格代码集扩展了这一点。虽然PolarQuant共享Hadamard变换的使用，但我们的方法在两个关键方面有所不同：（1）QuIP#对*整个*权重矩阵列（块间）应用旋转，而PolarQuant*在*每个128个元素的块内应用旋转（块内）；以及（2）QuIP#对约束最坏情况误差使用不相干，而PolarQuant使用归一化和旋转将分布转换为高斯。

### 2.6QuaRot和SpinQuant

QuaRot** **[[2](https://arxiv.org/html/2603.29078v1#bib.bib6 "QuaRot：旋转LLM中的无离群值4位推理")]表明将Hadamard旋转应用于隐藏状态、激活和KV缓存可以在不更改模型输出的情况下删除离群值，从而实现无离群值的4位推理。旋转量[[12](https://arxiv.org/html/2603.29078v1#bib.bib7 "SpinQuant：通过学习旋转进行LLM量化")]进一步表明，在零射击任务中，*学习的*旋转矩阵比固定的哈达马德旋转高出16分。这两种方法都应用*层之间的*旋转（需要图形手术将旋转吸收到相邻的线性层中），而PolarQuant独立地在每个权重张量*块内*应用旋转，不需要修改模型图，并允许作为一个简单的预处理步骤使用。

### 2.7涡轮量子

涡轮量子[[3](https://arxiv.org/html/2603.29078v1#bib.bib5 "TurboQuant：具有接近最佳失真率的在线矢量量化")]在推理过程中将极量化框架应用于KV缓存压缩，使KV向量归一化，应用随机旋转，并用最佳重心量化产生的近高斯坐标。TurboQuant证明了信息理论的下限，并实现了接近最佳的失真率。我们的工作将这种方法从KV缓存改编为*重量*压缩，展示了其作为下游INT4量化器的预处理步骤的有效性，并提供了第一个消融量化旋转（98%）与Q5最佳重心（2%）的单个贡献。

### 2.8劳埃德-马克斯量化

劳埃德-麦克斯算法[[13](https://arxiv.org/html/2603.29078v1#bib.bib8 "PCM中的最小二乘量化")，[14](https://arxiv.org/html/2603.29078v1#bib.bib9 "最小失真量化")]计算给定源分布的MSE最优標量量化器。对于一个来源X密度f（X），最佳量化器与L水平满足两个条件：（1）每个重心C我等于有条件的期望E[X∣X∈ℛ我]，哪里ℛ我是沃罗诺伊地区C我；以及（2）区域之间的边界是相邻重心的中点。对于标准正态分布，这些条件产生一个具有唯一解的方程组[[9](https://arxiv.org/html/2603.29078v1#bib.bib19 "量化")]，我们在第3[节](https://arxiv.org/html/2603.29078v1#S3 "3 方法‣ PolarQuant：通过哈达马德旋转进行LLM压缩的最佳高斯权量化")中利用了。

## 3方法

### 3.1一览表

PolarQuant量化一个权重张量W∈ℝm×N分四个阶段：

1. **1.**
   **块分解和归一化。**扁平W并划分成块{B我}我=1N大小D（我们使用D=128）。提取ℓ2规范R我=‖B我‖2并正常化：B^我=B我/R我。
2. **2.**
   **哈达马德轮换。**应用D×D归一化的沃尔什-哈达姆矩阵：B~我=HDB^我。旋转后，每个坐标大约是N（0，1/D）。
3. **3.**
   **缩放和量化。**缩放单位方差：Z我=D⋅B~我，所以Z我，J∼N（0，1）。将每个元素量化到最近的Lloyd-Max重心C∗=阿格分钟Ck⁡|Z我，J−Ck|。
4. **4.**
   **存储。**存储量化代码（int8）、每块规范（fp16）和重心表（fp32，全球共享）。

去量化是完全相反的：从代码中查找重心，缩放1/D，应用逆Hadamard旋转（因为HD−1=HD），并按存储的规范进行缩放R我。

### 3.2为什么哈达玛旋转会产生高斯坐标

我们现在为旋转坐标的高斯性提供了理论上的理由。

###### 定义3.1（规范化的沃尔什-哈达马矩阵）。

**沃尔什-哈达姆矩阵**HD**秩序的**D=2k**递归定义：**

|  | H1=[1]，H2D=12[HDHDHD−HD]。 |  | **(2)** |
| - | ---------------------------- | - | ------------- |

**这个矩阵是正交的（**HDHD⊤=我D**）和对称（**HD=HD⊤**），因此自我逆（**HD−1=HD**）。**

###### 命题3.2（旋转坐标的高斯性）。

**让**B^∈ℝD**是均匀分布在单位球面上的随机向量**SD−1**。让**HD**是归一化的Walsh-Hadamard矩阵。然后对于每个坐标**J**，旋转的元素**B~J=（HDB^）J**满足：**

|  | D⋅B~J→DN（0，1）作为D→∞。 |  | **(3)** |
| - | ----------------------------- | - | ------------- |

###### 证明草图。

因为HD是正交的，B~=HDB^也均匀地分布在SD−1。一个均匀点的每个坐标SD−1有邊際分佈[[9](https://arxiv.org/html/2603.29078v1#bib.bib19 "量化")]：

|  | B~J∼1D⋅贝塔（12，D−12）1/2⋅拉德马赫， |  | **(4)** |
| - | ----------------------------------------- | - | ------------- |

根据球体投影的中心极限定理，它收敛到N（0，1/D）作为D→∞。为了D=128，近似值非常好：Kolmogorov-Smirnov统计量在旋转的LLM权重坐标的经验分布和N（0，1/D）通常低于0.01。∎

###### 备注3.3**。**

**实际论点甚至比渐近论点更有说服力。真正的LLM权重块，在归一化后，在球面上并不完全均匀，但它们的经验分布足够接近，高斯近似值很好。哈达玛德旋转使坐标** *同质化* **：即使原始块有几个大元素（离群值）和许多小元素，旋转将能量分散到所有坐标，产生近高斯边际分布。**

### 3.3标准正态分布的劳埃德-马克斯重心

鉴于旋转和缩放坐标Z我，J∼N（0，1），MSE最优量量化器与L=2B水平是通过Lloyd-Max算法获得的。

###### 定理3.4（Lloyd-Max最优条件）。

**对于连续的来源**X**密度**f（X）**，具有电平的MSE优化量化器**{C1，...，CL}**和决策界限**{t0，t1，...，tL}**（哪里**t0=−∞**，**tL=+∞**）满足：**

|   | C我 | =E[X∣t我−1<X≤t我]=∫t我−1t我Xf（X）DX∫t我−1t我f（X）DX， |   | **(5)** |
| - | --- | -------------------------------------------------------------- | - | ------------- |
|   | t我 | =C我+C我+12，我=1，...，L−1。                                 |   | **(6)** |
| - | -   | -                                                              | - | -             |

对于标准正态分布f（X）=φ（X）=12πE−X2/2，重心条件（5）简化为闭合式表达式：

|  | C我=φ（t我−1）−φ（t我）Φ（t我）−Φ（t我−1）， |  | **(7)** |
| - | ---------------------------------------------------- | - | ------------- |

地点φ（⋅）和Φ（⋅）分别是标准的普通PDF和CDF。这源于身份∫一个BXφ（X）DX=φ（一个）−φ（B）。

Lloyd-Max算法在方程（7）和（6）之间迭代，从均匀间隔的边界开始，并在MSE中单调地收敛。在实践中，在50次迭代内实现与机器精度的融合；为了安全起见，我们使用100次迭代。

###### 命题3.5（对称性）。

**对于标准正态分布，Lloyd-Max量化器与**L=2B**水平围绕零对称：**C我=−CL+1−我**为了所有**我**。**

###### 证明。

标准正态密度是对称的（φ（X）=φ（−X）），劳埃德-麦克斯定点方程保持了这种对称性。根据固定点的唯一性，解必须是对称的。∎

[提案̃**3.5**](https://arxiv.org/html/2603.29078v1#S3.Thmtheorem5 "命题3.5（对称性）。‣ 3.3 标准正态分布的Lloyd-Max重心 ‣ 3 方法 ‣ PolarQuant：通过LLM压缩的Hadamard旋转进行最佳高斯权量化")将重心表的存储要求减半，并简化了实施。

###### 命题3.6（MSE优于Absmax）。

**让**MSELM**和**MSE腹肌**分别表示Lloyd-Max和absmax量化器的均方量化误差，对于**N（0，1）**来源在**B**位元。然后：**

|  | MSELMMSE腹肌≤0.46在B=3。 |  | **(8)** |
| - | ------------------------- | - | ------------- |

与absmax相比，54%的MSE降低是PolarQuant的核心优势。改进的产生是因为absmax将量化水平均匀地放在[−α，α]，在高斯密度低的尾部浪费分辨率，而劳埃德-马克斯在高密度中央区域集中水平。

### 3.5极量+AWQ

AWQ** **[[11](https://arxiv.org/html/2603.29078v1#bib.bib2 "AWQ：用於裝置上LLM壓縮和加速的啟動感知重量量化")]和PolarQuant在正交轴上运行：AWQ重新缩放*通道*以保护激活敏感权重，而PolarQuant则归一化和旋转块以实现最佳量化。他们的组合很简单：

1. **1.**
   计算每通道的AWQ比例S来自校准数据。
2. **2.**
   将权重乘以天平：W′=W⋅诊断（S）。
3. **3.**
   将PolarQuant应用于W′。
4. **4.**
   在去量化时，应用逆AWQ：W^=W^′⋅诊断（S−1）。

这种组合减少了困惑的delta+0.52（仅PolarQuant Q5）只是+0.06（PolarQuant Q5 + AWQ），如第4[节](https://arxiv.org/html/2603.29078v1#S4 "4个实验‣PolarQuant：通过哈达马德旋转进行LLM压缩的最佳高斯权量化")所示。

### 3.6PolarQuant作为INT4推理的预处理

这项工作的一个关键贡献是表明PolarQuant可以作为 *预处理步骤* ，提高下游INT4量化的质量。管道是：

|  | W→极量Q5W^PQ→脱量BF16W^BF16→torchao INT4W^INT4。 |  | **(9)** |
| - | --------------------------------------------------- | - | ------------- |

直觉如下。带有Lloyd-Max重心的PolarQuant Q5产生去量化的重量W^BF16与原来的MSE较低W比absmax Q5去量化。当torchao随后使用分组absmax将此中间表示重新量化为INT4时，它从*更好的*近似开始W。计算的分組absmax尺度因子W^BF16更能代表真正的权重分布，因为PolarQuant已经消除了离群值引起的失真。

这不是传统意义上的双重量化（两个量化步骤都是有损的，误差是复合的）。相反，PolarQuant作为一个 *去噪步骤* ：Q5量化和去量化有效地将权重投射到更符合后续absmax INT4量化的表示上。

**基线。**我们比较：（1）FP16，全精度参考；（2）torgao INT4** **[[16](https://arxiv.org/html/2603.29078v1#bib.bib16 "Torchao：PyTorch架构优化")]具有分组absmax量化（组大小128）；以及（3）BitsAndBytes NF4** **[[7](https://arxiv.org/html/2603.29078v1#bib.bib3 "QLoRA：量化语言模型的高效微调")]，NormalFloat 4位格式。

### 4.4消融研究

[表̃3](https://arxiv.org/html/2603.29078v1#S4.T3 "在4.4消融研究中‣4个实验‣PolarQuant：通过哈达马德旋转进行LLM压缩的最佳高斯重量量化")分解了PolarQuant的两个组成部分的贡献。结果揭示了一个引人注目的发现：**仅哈达马德旋转就占了98%的改善**，将困惑程度从6.90降低到6.40（Δ=−0.50）。Lloyd-Max重心在Q5只提供微量的额外收益（Δ=−0.01），将困惑度带到6.39。

在较低的位宽度（例如，B=2或者B=3），在少数可用的重心必须谨慎放置的地方，Lloyd-Max重心将贡献更大的改善部分。这与[命题̃**3.6**](https://arxiv.org/html/2603.29078v1#S3.Thmtheorem6 "命题3.6（MSE优于Absmax）。‣ 3.3 标准正态分布的Lloyd-Max重心 ‣ 3 方法 ‣ PolarQuant：通过LLM压缩的Hadamard旋转进行最佳高斯权量化")一致，该命题显示第三季度MSE下降了54%。

## 5分析

### 5.1为什么PolarQuant改进下游INT4

这项工作的核心发现是，当torchao INT4重新量化PolarQuant Q5去量化权重时，比直接的absmax INT4产生更好的复杂度。我们提供以下解释。

Torchao使用分组absmax INT4量化，组大小为128。对于每组128个权重，它计算S=马克斯⁡|WJ|并使用此比例尺将所有值映射到4位整数。这种量化的质量在很大程度上取决于每个组的 *动态范围* ：具有离群值的组具有大S以及大多数价值观的解决能力差。

正如我们的消融（第4.4[节](https://arxiv.org/html/2603.29078v1#S4.SS4 "4.4 消融研究‣4个实验‣PolarQuant：通过Hadamard旋转进行LLM压缩的最佳高斯权重量化")）所揭示的那样，主要机制是哈达玛旋转，占质量改进的98%。旋转将重量分布从重尾、非均匀的形状转变为近似高斯形状。在高斯制度中，每个128元素群的动态范围都紧密集中在2ln⁡128≈3.1标准差，因此分组的absmax比例因子密切跟踪真实数据范围。如果没有旋转，离群值权重会增加absmax秤，浪费尾部的分辨率。

PolarQuant Q5去量化权重已经经历了这种分布归一化。当torchao随后重新量化为INT4时，它遇到了跨组更均匀的权重分布，离群值更少，动态范围更一致。结果是，每个INT4组都能更好地利用其16个可用级别。

从本质上讲，PolarQuant充当 *分布正则器* ：Hadamard旋转使权重分布均质化，产生更易于均匀（absmax）量化的群。

### 5.2双重量化悖论

人们可能会期望在torchao INT4之前在*较低*的位宽（例如Q3）下应用PolarQuant会产生更好的结果，因为分布正则化会更强。令人惊讶的是，事实恰恰相反：

* **•**
  混合位PolarQuant（Q3用于门/向上投影）+torchao INT4：PPL 7.25（ *更差* ）
* **•**
  Uniform PolarQuant Q5 + torchao INT4：PPL 6.56（ *更好* ）

解释是Q3去量化*太*有损：只有8个重心，去量化的权重会丢失torchao的INT4量化器无法恢复的关键精细信息。最佳操作点要求PolarQuant预处理保留足够的信号，供下游量化器利用。在Q5（32个重心），去量化的权重保留了足够的信息，以便torchao产生高质量的INT4表示。

这确立了级联量化的原则： *预处理量化器必须在足够高的位元宽度下运行，以保留下游量化器所需的信息。* 在我们的设置中，Q5是Q5的经验上最优的中间位宽→INT4级联。

### 5.3计算间接成本

PolarQuant去量化为RTX PRO 6000 Blackwell的模型加载时间增加了大约8秒。这是一次性费用。在推理时，计算路径与标准FP16或INT4推理相同，额外的开销为零。去量化过程中的主要成本是批处理矩阵乘法H128B^我对于每个区块；我们的实现使用PyTorch的**torch.matmul**，它实现了25×通过利用优化的cuBLAS GEMM内核，执行速度比天真快的Walsh-Hadamard变换（FWHT）实现更快。

## 6结论

我们介绍了PolarQuant，这是一种简单有效的重量量化方法，通过利用哈达玛旋转重量块的高斯结构来实现近乎无损的压缩。我们的消融表明，Hadamard旋转是基本组成部分，占Q5质量改进的98%。我们的关键贡献是：

1. **1.**
   **近乎无损的压缩。**PolarQuant Q5在Qwen3.5-9B上实现了困惑度6.39（Δ=+0.02来自FP16）没有任何校准数据，每个块只需要一个Hadamard矩阵乘法。
2. **2.**
   **轮换是关键。**我们的消融表明，仅哈达马德旋转就占了质量改进的98%（PPL 6.90→6.40），而劳埃德-马克斯重心在Q5只提供微不足道的额外收益。这一发现将该方法简化为其基本组成部分：确定性正交旋转。
3. **3.**
   **改进了INT4推理。**当用作预处理步骤时，PolarQuant Q5 + torchao INT4在相同的吞吐量（43.1 tok/s）和接近相同的内存（6.5 GB）下，直接absmax INT4实现了6.6.68的困惑度。
4. **4.**
   **跨平台部署。**PolarQuant在NVIDIA GPU和Apple Silicon上运行，在4.8 GB的Mac mini M4上实现19.7 tok/s。

PolarQuant与任何下游量化器（torchao、GGUF、MLX）兼容，其核心算法不需要校准数据（AWQ是唯一使用校准的组件）。该方法不增加运行时开销，在模型加载时只有几秒钟的一次性去量化成本。

**限制和未来工作。**PolarQuant假设Hadamard旋转的重量块是由i.i.d.很好地近似的。高斯，可能不适用于所有建筑。当前的实现没有利用块间相关性。未来的工作可以将PolarQuant扩展到激活量化，用高斯代码集探索矢量量化，并调查级联量化管道的理论极限。

代码和模型在[https://github.com/caiovicentino/eoq-quantization](https://github.com/caiovicentino/eoq-quantization)和[https://huggingface.co/caiovicentino1](https://huggingface.co/caiovicentino1)上公开提供。

## 附录B混合位元分配策略

受Unsloth Dynamic 2.0的启发，我们研究了一种混合位策略，该策略根据量化灵敏度为不同的张量类型分配不同的位宽：

表6：按张量类型划分的混合位分配。

| **张量类型**         | **位元** | **理由**             |
| -------------------------- | -------------- | -------------------------- |
| **MLP门/向上投影**   | **3**    | **量化最稳健**       |
| **MLP向下投影**      | **4**    | **稍微更敏感一点**   |
| **注意Q/K/V**        | **5**    | **中等敏感性**       |
| **注意O/输出proj**   | **6**    | **高灵敏度**         |
| **嵌入**             | **5**    | **安全的**           |
| **LM头**             | **6**    | **几乎无损**         |
| **规范、偏见、路由** | **FP16** | **必须保持完全精确** |

这实现了一个∼3.7位平均值，在较小的存储中，质量可与统一的Q5相当。然而，如第5.2节所述，使用此混合位方案作为torchao INT4的预处理会降低质量（PPL 7.25），因为Q3层丢失了太多信息。因此，混合位策略只建议用于直接PolarQuant推理（没有下游重新量化）。
